# Codex Phase 1 Gate Review — Architecture Hygiene

> codex-cli read-only review of Phase 1 (lazy root_agent, runner<->pipeline cycle break, output keys). It ran the no-cycle smoke import + ruff independently.

**Findings**

- **[MEDIUM] Task 1.1 ISSUE: `app.agent.__getattr__` does not cache the built `App` as a single object.**  
  Evidence: [app/agent.py:38](<app/agent.py:38>) builds on either `"app"` or `"root_agent"` access, but [app/agent.py:42](<app/agent.py:42>) caches only the requested name. Probe showed `hasattr(app.agent, "app")` then `app.agent.root_agent` built twice: `final_count 2`, and `root_same_as_app False`.  
  ADK itself checks `app` first, so loader discovery is okay, but the stated “App builds at most once” contract is false. There is also no lock around build/cache, so concurrent first access can race and double-build.

- **[LOW] Task 1.1 caveat: no DB write is fixed, but literal “NO disk I/O” is not true for imports like `app.runner`.**  
  `app.runner` imports `build_pipeline` at [app/runner.py:20](<app/runner.py:20>), which imports modules that read prompt files at import time, e.g. [app/pipeline/processing.py:25](<app/pipeline/processing.py:25>), [app/pipeline/critic.py:20](<app/pipeline/critic.py:20>), [app/pipeline/digest_editor.py:18](<app/pipeline/digest_editor.py:18>). The SQLite creation side effect is gone.

**Task Verdicts**

- **Task 1.1 — ISSUE.**  
  `app/__init__.py` no longer imports `.agent`, so package import no longer builds storage. A probe with `SQLITE_PATH=data/__phase1_import_probe_should_not_exist.db` printed `before False` / `after False` after importing `app.core.domain`, `app.runner`, and `app.agent`. ADK loader discovery is supported: its source checks `hasattr(module_candidate, "app")` first in `.venv/.../agent_loader.py:125`, which triggers module `__getattr__`. No live code references `from app import app` or `from app.agent import app/root_agent`; only docs still describe the old eager export.

- **Task 1.2 — AGREE.**  
  The cycle is gone: `runner` imports `build_pipeline` at top level, `agents` imports wiring at [app/pipeline/agents.py:47](<app/pipeline/agents.py:47>), and `wiring` imports neither `runner` nor `build_storage` at [app/pipeline/wiring.py:18](<app/pipeline/wiring.py:18>). Direct smoke passed: `import app.runner; import app.pipeline.agents; print('ok')` -> `ok`.  
  Monkeypatch targets still work: `runner.rss/newsapi/scrape` are the same module objects used by `wiring`, and `runner.markdown/html_render` are the same render modules used by `RenderAgent`. `app.pipeline.agents._collect` still resolves and is read by `build_pipeline` at [app/pipeline/agents.py:471](<app/pipeline/agents.py:471>). `run_id` is removed from the live signature and call sites; remaining hits are stale docs.

- **Task 1.3 — AGREE.**  
  Backend writes exactly `md`, `xlsx`, `html` at [app/pipeline/agents.py:438](<app/pipeline/agents.py:438>). Frontend reads exactly `html`, `xlsx`, `md` at [frontend/components/digests/output-links.tsx:11](<frontend/components/digests/output-links.tsx:11>). Backend contract test asserts the exact output key set at [tests/integration/test_pipeline_tree.py:109](<tests/integration/test_pipeline_tree.py:109>), and the frontend test proves all three backend keys render labels/basenames.

**Verification**

- `PYTHONDONTWRITEBYTECODE=1 .venv/bin/python -c "import app.runner; import app.pipeline.agents; print('ok')"` -> `ok`
- `.venv/bin/ruff check --no-cache app tests` -> `All checks passed!`
- Exact `uv run ...` commands could not run because the read-only sandbox blocks uv cache initialization under `~/.cache/uv`.

## Disposition
- [MEDIUM] __getattr__ per-name double-build: FIXED — build once, cache both app+root_agent under a lock; test asserts app.root_agent is root_agent.
- [LOW] 'no disk I/O' wording: softened to 'no DB creation / disk writes' (prompt-file reads happen via the app.runner import chain, not via importing app.agent).
- Tasks 1.2 + 1.3: AGREE, no action.
