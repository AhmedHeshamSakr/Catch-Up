## Findings

- ISSUE [HIGH] `docs/superpowers/specs/2026-05-23-adk-catchup-agent-design.md:208-220`  
  §13/§14 still overclaim resilience/cost controls. There is no timeout on the search-grounding ADK call (`app/services/search.py:69-78`), YouTube transcript fetch has no explicit timeout (`app/services/youtube.py:120-173`), `run_timeout` is optional and default `None` (`app/core/config.py:97-101`), structured-output parsing has no separate “one retry” (`app/llm/parse.py:88-111`, `app/pipeline/processing.py:153-157`), and I found no RSS conditional GET, grounding cache, or HTTP response cache in the collectors.

- ISSUE [MED] `docs/superpowers/specs/2026-05-23-adk-catchup-agent-design.md:203-205`  
  “Retries ... on LLM calls” is too broad. The custom loop exists for `run_agent_text` (`app/llm/runtime.py:83-109`) and settings exist (`app/core/config.py:93-96`), but search grounding is also an LLM/ADK call and bypasses that retry/timeout path (`app/services/search.py:69-78`). Say “structured `app/llm.run_agent_text` calls” if that is the intended scope.

- ISSUE [MED] `docs/superpowers/specs/2026-05-23-adk-catchup-agent-design.md:90`  
  The SOLID bullet still says “dependency inversion at every swap-point (interfaces in core, implementations injected),” which contradicts the newly honest §6/§12 claim that only storage has a real port. Actual port evidence: `app/core/ports/storage.py:8-41`; scheduler and LLM are direct modules (`app/services/scheduler.py:18-36`, `app/llm/runtime.py:36-58`).

- ISSUE [MED] `docs/superpowers/specs/2026-05-23-adk-catchup-agent-design.md:308-310`  
  §21’s file tree omits a real major top-level module: `app/run_trigger.py`, the shared single-flight run starter used by both API and scheduler (`app/run_trigger.py:1-42`, `app/api/app.py:316-325`, `app/api/app.py:384-388`). It also omits `feed_discovery.py` and `youtube_resolve.py`, which back `POST /api/sources/resolve` (`app/api/app.py:327-355`).

- ISSUE [LOW] `docs/superpowers/specs/2026-05-23-adk-catchup-agent-design.md:215-216`  
  Endpoint names are missing the `/api` prefix. Actual routes are `POST /api/runs` and `POST /api/sources/resolve` because the router has `prefix="/api"` (`app/api/app.py:183`) and the decorators are `/runs` and `/sources/resolve` (`app/api/app.py:316-328`).

- ISSUE [HIGH] `docs/superpowers/specs/2026-05-23-adk-catchup-agent-design.md:280`  
  §19 still tells production Cloud Scheduler to call `/api/digests/run`, which does not exist. The real trigger is `POST /api/runs` (`app/api/app.py:316-325`), and the fixed wording elsewhere already says that.

- ISSUE [MED] `docs/superpowers/specs/2026-05-23-adk-catchup-agent-design.md:107-123`  
  The domain model block still claims removed fields: `NewsItem.org_id`, `NewsItem.user_id`, `NewsItem.language`, and `DigestRun.org_id`. Actual models do not have them (`app/core/domain.py:129-180`). This directly conflicts with Phase 4’s “drops & deletes” log (`docs/BUILD-LOG.md:281`).

- ISSUE [MED] `docs/superpowers/specs/2026-05-23-adk-catchup-agent-design.md:165` and `:287`  
  The spec still says “ADK eval.” Actual guidance and code use the custom enrichment harness (`CLAUDE.md:21-24`, `scripts/eval_enrichment.py:1-10`, `docs/eval/README.md:140`).

- ISSUE [LOW] `docs/ADK-GUIDE.md:56` and `:144`  
  The guide still has stale phrasing: structured agents “use ... `output_key=...`” even though Phase 4 removed output keys, and §4d says `app/agent.py` exports `app = App(...)` directly even though the App is lazy via `__getattr__` (`app/agent.py:42-53`). The App row at `docs/ADK-GUIDE.md:18` is accurate; this later text just contradicts it.
