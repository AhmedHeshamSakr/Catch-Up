# Codex Phase 0 Gate Review — Security + CI

> codex-cli read-only review of the Phase 0 diff (fail-closed API, SSRF cap, CSRF, /feedback auth, CI). It independently ran ruff + tests/unit/test_net.py (14 passed).

**Findings**

1. **HIGH ISSUE:** `app.fast_api_app` still exposes ADK mutating routes without API-key auth. The guard at [app/fast_api_app.py](<app/fast_api_app.py:49>) only requires `API_KEY` to exist; it does not protect the ADK app created at [app/fast_api_app.py](<app/fast_api_app.py:74>). `/api/*` is protected through `register_product_routes`, and `/feedback` is protected at [app/fast_api_app.py](<app/fast_api_app.py:102>), but ADK routes such as `/run`, sessions, evals, and builder routes remain on the same public app. That is a deployed-surface bypass unless Cloud Run/IAM blocks unauthenticated traffic externally.

2. **MEDIUM ISSUE:** `safe_get` does not make the size cap strict for all streamed bodies. At [app/services/net.py](<app/services/net.py:142>), `iter_bytes()` is called without a fixed `chunk_size`, and [app/services/net.py](<app/services/net.py:143>) appends the chunk before checking the cap. A lying/chunked response, especially decoded content, can allocate one over-large chunk before the guard fires. Redirects are revalidated and streams are closed, but the cap should check projected size before extend and ideally bound chunking/decoded amplification.

3. **MEDIUM ISSUE:** `create_app()` can only check `settings.app_host`, not the actual ASGI bind host. The CLI path correctly checks `--host` at [app/cli.py](<app/cli.py:36>), but direct `uvicorn app.api.app:create_app --factory --host 0.0.0.0` with default `APP_HOST=127.0.0.1` passes [app/api/app.py](<app/api/app.py:371>). Repo-owned `scripts/run.sh` binds loopback and Docker uses `fast_api_app`, so this is a residual direct-factory bypass, not the default path.

4. **LOW ISSUE:** Coverage is incomplete. No test imports `app.fast_api_app` to prove the import guard or `/feedback` 401/429 behavior; [tests/integration/test_deploy_surface.py](<tests/integration/test_deploy_surface.py:5>) explicitly avoids that import. `safe_get` tests cover cap branches but do not assert close-on-redirect/raise/return, and the Content-Length test builds the oversized mock body before `safe_get` sees it.

**Item Verdicts**

1. **Fail-closed:** ISSUE. Loopback classification is correct for `127.0.0.1`, `::1`, `localhost`; `0.0.0.0` and LAN IPs are non-loopback. CLI and Docker paths are mostly covered, but direct ASGI factory bind and ADK route auth remain gaps.

2. **SSRF size cap:** ISSUE. Redirect validation and stream closing look correct; the cap is not strict against oversized decoded/chunked chunks.

3. **CSRF:** AGREE. The product mutating routes are covered: `PUT /sources`, `PUT /watchlist`, `POST /runs`, `POST /sources/resolve`; settings writes already use `_require_local_write`. It correctly no-ops when `api_key` is set. Minor caveat: allowlist comparison is hostname-only, not exact scheme/port origin.

4. **`/feedback`:** AGREE for that route’s auth + rate limit, ISSUE for missing tests and the broader ADK app remaining unauthenticated.

5. **CI:** AGREE. `uv sync --frozen`, `ruff check app tests scripts`, pytest paths, `npm run test`, `npm run build`, and gitleaks with full history are coherent and should gate if the workflow is required.

6. **Tests:** ISSUE. The new product-route CSRF tests are meaningful, but Phase 0 lacks negative/positive tests for fast_api_app, `/feedback`, CLI `--host`, redirect closing, and strict streamed cap behavior.

**Verification**

- `ruff check --no-cache` on touched backend/test files: passed.
- `pytest -q -s -p no:cacheprovider tests/unit/test_net.py`: `14 passed`.
- Full integration tests were not run because the sandbox has no writable temp directory; `pytest` capture failed until disabled, and `tmp_path` tests would fail for environment reasons.

## Disposition
- #2 (streamed-cap strictness): FIXED — iter_bytes(chunk_size=65536) + check-before-extend (commit follows).
- #1 (ADK-native routes unauthenticated) + #3 (direct --factory bind bypass): CARRIED to Phase 3 — the public product image runs app.web_app (create_app: console + /api only, no ADK routes, fully key-guarded); fast_api_app is the Agent Engine surface and is documented as requiring Cloud Run IAM/IAP. web_app.py will require API_KEY explicitly.
- #4 (coverage): accepted — fast_api_app cannot be imported in the offline suite by design.
